## N皇后题目介绍

题目详情:
按照国际象棋的规则，皇后可以攻击与之处在同一行或同一列或同一斜线上的棋子。
**n 皇后问题** 研究的是如何将 `n` 个皇后放置在 `n×n` 的棋盘上，并且使皇后彼此之间不能相互攻击。
给你一个整数 `n` ，返回所有不同的 **n 皇后问题** 的解决方案。
每一种解法包含一个不同的 **n 皇后问题** 的棋子放置方案，该方案中 `'Q'` 和 `'.'` 分别代表了皇后和空位。

## 题目解答
```java
class Solution {  
    int[] a = new int[10];//下标表示所处行数,数组的值表示所处列数  
    List<List<String>> res = new ArrayList<>();  
  
    public List<List<String>> solveNQueens(int n) {  
        dfs(n, 0);  
        return res;  
    }  
  
    void dfs(int n, int r) {  
        if (r >= n) {  
            addRes(n);  
            return;  
        }  
  
        //摆好皇后  
        for (int i = 0; i < n; i++) {  
            a[r] = i;  
            if (check(r)){  
                dfs(n, r + 1);  
            }  
        }  
    }  
  
    boolean check(int r) {//用于剪枝操作  
        for (int i = 0; i < r; i++) {  
            if (a[i] == a[r] || Math.abs(a[i] - a[r]) == Math.abs(i - r)) {  
                return false;  
            }  
        }  
        return true;  
    }  
  
    void addRes (int n){  
        //创建一行的模板  
        char[] s = new char[n];  
        Arrays.fill(s, '.');  
        List<String> list = new ArrayList<>();  
        for (int i = 0; i < n; i++) {  
            s[a[i]] = 'Q';  
            list.add(new String(s));  
            s[a[i]] = '.';  
        }  
        res.add(list);  
    }  
}
```

## 本题思路讲解:
1. 先把皇后放置在棋盘上(比如先都放在第一列,每行一个皇后)
![600](https://images.sueryeto.cn/images/20260401002711521.webp)
2. 想象一下为满足题目条件,我们应该怎么放皇后?我们可以把最后一行(第三行)的Q在该行进行移动,直到满足题目条件,但是值得注意的是,我们一开始对于皇后的排列就完全不符合规则(4个Q在同一条线上),所以为了避免过多的无效操作,我们应该对它们进行剪枝操作(即check方法)
3. 对于皇后的移动,我们使用DFS的思想:**一行一行地放皇后，每放一个就检查会不会打架，不打架就继续放下一行，打架就换一列；全部放完就保存答案，放不下去就回头（回溯）。** 
4. 对于回溯:我们可以采取全局数组表示皇后位置的方法规避回溯的需求(即最上面的`int[] a = new int[10];`),下标表示所处行号,值表示所处的列号
- 循环 `i++` 会自动试下一列
- 函数执行完自动回到上一层
- `a[r] = i` 下一轮会直接覆盖旧值
___
## 核心代码拆解:
```java
void dfs(int n, int r) { 
// 1. 递归终止条件：所有行都放完皇后了 
	if (r >= n) { 
		addRes(n); // 保存这一组正确答案 
		return; // 结束当前这层递归
	} 
	
	// 2. 在第 r 行，尝试每一列：从第0列 → 第1列 → 第2列…… 
	for (int i = 0; i < n; i++) {
		a[r] = i; // 把皇后放在：第 r 行、第 i 列
	
	// 3. 检查：这个位置和之前放的皇后冲突吗？ 
		if (check(r)){ 
			dfs(n, r + 1); // 不冲突 → 去下一行放皇后
		}
		
	// 4. 回溯：循环自动继续，尝试下一列
	}
}
```
---
### 提取重点:
1. `r` 是什么？
`r = 当前要放皇后的行号`

- 从 `r=0` 开始（第 0 行）
- 放完一行，`r+1` 进入下一行
- 当 `r >= n`，说明**所有 n 行都放完了** → 得到一个合法答案

2. 循环 `for (int i=0; i<n; i++)` 干什么？
**在当前第 r 行，从左到右一列一列试放皇后**

- i = 尝试放的列号
- 不行就退回，试下一列

2. `check(r)` 是剪枝
**只和已经放好的皇后检查冲突**

- 同列 ❌
- 对角线 ❌
- 同行 ✅（天然不会，因为一行只放一个）
___
## 算法模拟(AI演示):
### 初始状态

n=4，从第 0 行开始放：`dfs(4, 0)`

---
### 第一层递归：r = 0（第 0 行）

进入循环，尝试列 0、1、2、3：

1. `a[0] = 0` → 第 0 行第 0 列
2. `check(0)` → 没冲突
3. **进入下一行：`dfs(4, 1)`**

---

### 第二层递归：r = 1（第 1 行）

尝试列 0、1、2、3：

- `a[1]=0` → 和第 0 行同列 ❌
- `a[1]=1` → 对角线冲突 ❌
- `a[1]=2` → 检查通过 ✅
- **进入下一行：`dfs(4, 2)`**

---

### 第三层递归：r = 2（第 2 行）

尝试每一列：

- 列 0：冲突
- 列 1：冲突
- 列 2：冲突
- 列 3：冲突
    
    **全部都冲突！没有能放的位置！**

👉 **这时候发生【回溯】**

循环结束，自动退回上一层（r=1）！

---

### 回到第二层递归 r=1，继续循环！

刚才试了列 2 不行，现在继续试：

- `a[1]=3` → 检查通过 ✅
    
    👉 **进入下一行：`dfs(4, 2)`**

---

### 第三层递归 r=2

继续尝试列 0、1、2、3……

不行就回溯，行不行就再回上一行……

**整个过程就是：尝试 → 检查 → 前进 / 回溯**

---

### 什么时候算成功？

当 `r = 4`（n=4）

`r >= n` 条件成立 → 说明**4 行都放好了**

→ 调用 `addRes` 保存答案

→ `return` 退回，继续找其他解法

## 一句话总结 DFS 运行规律（重要）

1. **到第 r 行**
2. **从左到右试放皇后**
3. **不冲突就去下一行（r+1）**
4. **冲突就换一列**
5. **所有列都不行 → 退回上一行（回溯）**
6. **所有行放完 → 保存答案**
___
## 关键点回顾

1. **`r` = 当前要放皇后的行**
2. **`for i` = 在当前行尝试每一列**
3. **`check(r)` = 剪枝，只检查已放好的皇后**
4. **不冲突 → 递归到下一行 `r+1`**
5. **冲突 / 循环结束 → 自动回溯到上一行**
6. **`r >= n` → 找到答案**

```java
void addRes (int n){
	char[] s = new char[n];
	Arrays.fill(s, '.'); // 初始化为全点 
	//用来保存一整个棋盘
	List<String> list = new ArrayList<>();
	
	for (int i = 0; i < n; i++) { 
		s[a[i]] = 'Q'; // 放皇后 
		list.add(new String(s));// 转字符串存入 
		s[a[i]] = '.'; // 恢复为点
	}
	res.add(list); // 把这一种解法加入总结果
}
```
## 个人理解:
对于DFS以及它深层的概念--递归,我目前对它依旧闻风丧胆.但是经过刚刚的总结我逐渐也有了些自己对于DFS的想法,就拿这道题举例(用check()方法确定合法性),说白了就是先确定第零行的位置(即列号),然后调用确定第零行位置的方法去找后面一行(即第一行)的位置是否合法,要是合法就继续调用找本行的方法去找下一行;要是不合法就进行剪枝,表明不需要在此基础上去找其他行的合法性了,而是回到本行还没确定位置的时候去判断放置在下一列是否合法,以此类推直到达成了停止条件,此时表明一个正确棋盘已经生成好了,并将它添加到res二维动态字符串数组里.

交给AI润色之后的理解:
N 皇后 DFS 执行流程：

1. 从第 0 行开始，尝试在每一列放置皇后；
2. 放置后检查是否与已放皇后冲突：
    
    - 不冲突：继续递归，去下一行放置皇后；
    - 冲突：剪枝，不再向下探索，直接尝试当前行的下一列；
    
3. 重复 “尝试 → 检查 → 递归 / 剪枝 → 回溯”；
4. 当所有行都放置完成，生成合法棋盘，加入结果集。
